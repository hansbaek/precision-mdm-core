1. 시스템 개요

T:MDM은 타이어 시험 기준·분류 마스터 데이터를 관리하고, 제품코드(mcode) 기반으로 수행해야 할 시험을 조회하는 웹 애플리케이션입니다.

  • 접속 방식: 웹 브라우저 (로그인 후 사용)
  • 인증: 아이디/비밀번호 로그인 (JWT 토큰 기반)
  • 접근 제어: 역할(Role) 기반으로 메뉴·기능 단위 권한을 제어

1.1 기본 주소

구분주소
웹 화면 (개발)http://localhost:3000
API 서버 (개발)http://localhost:4000
API 문서 (Swagger)http://localhost:4000/docs

운영 환경 주소는 시스템 관리자에게 문의하십시오.

2. 로그인 / 로그아웃

2.1 로그인

  1. 브라우저에서 시스템 주소로 접속하면 로그인 화면(/login)으로 이동합니다.
  2. 아이디와 비밀번호를 입력합니다.
  3. (선택) 아이디 저장을 체크하면 다음 접속 시 아이디가 자동 입력됩니다.
  4. 로그인 버튼을 클릭합니다.

로그인에 성공하면 본인에게 허용된 메뉴만 표시된 메인 화면으로 이동합니다. 아이디 또는 비밀번호가 틀리면 화면 하단에 오류 메시지가 표시됩니다.

2.2 최초 관리자 계정

시스템 최초 설치 시 기본 관리자 계정(admin)이 제공됩니다. 초기 비밀번호는 시스템 관리자(설치 담당자)에게 별도로 문의하십시오.

보안 주의: 최초 로그인 후, 관리자 화면에서 비밀번호를 반드시 변경(재설정)하십시오.

2.3 세션 유지

  • 로그인 상태는 브라우저에 토큰으로 저장되어, 새로고침해도 유지됩니다.
  • 토큰 유효기간(기본 8시간)이 지나면 자동으로 로그아웃되며 다시 로그인해야 합니다.

2.4 로그아웃

화면 우측 상단의 사용자 영역(이름/이니셜)을 클릭하면 메뉴가 열립니다. 로그아웃을 선택하면 로그인 화면으로 돌아갑니다.

3. 화면 구성

┌──────────┬─────────────────────────────────────────────┐ │ │ 헤더: 탭 · 검색 · 알림 · 언어 · 테마 · 사용자 │ │ 사이드바 ├─────────────────────────────────────────────┤ │ (모듈) │ │ │ │ 본문 (선택한 탭 화면) │ │ ├─────────────────────────────────────────────┤ │ │ 푸터: 저작권 · 약관 │ └──────────┴─────────────────────────────────────────────┘

3.1 사이드바 (모듈)

좌측 사이드바에는 본인에게 권한이 있는 모듈만 표시됩니다.

모듈설명
시험 항목 기준 마스터시험 기준 템플릿 관리 · 필요시험조회 · 분석 · 보고서
시험 분류 마스터시험 분류 체계 관리
재료 사양 / 차량 구성 / 데이터 감사(준비 중)
접근 권한 관리(관리자 전용) 역할·권한·사용자 관리
  • 상단의 ◀ / ▶ 버튼으로 사이드바를 접거나 펼 수 있습니다.
  • 모듈을 클릭하면 해당 모듈의 첫 번째 탭으로 이동합니다.

3.2 헤더

  • 탭: 현재 모듈에 속한 하위 탭. 권한이 있는 탭만 보입니다.
  • 검색(Ctrl+K): 커맨드 팔레트로 항목·탭을 빠르게 이동.
  • 언어 토글: 한국어 ↔ English.
  • 테마 토글: 라이트 ↔ 다크 모드.
  • 도움말(?): 본 매뉴얼을 새 탭으로 엽니다.
  • 사용자 영역: 이름·소속 표시, 클릭 시 로그아웃.
↑ 목차로

4. 시험 항목 기준 마스터

시험 항목 기준 마스터 모듈은 4개의 탭으로 구성됩니다.

4.1 기준정보 관리 (대시보드)

시험 기준 템플릿(TEMPLATE_STD_TEST_ITEM)을 조회·관리합니다.

  • 검색/필터: 제품라인·검색어·시장 코드로 필터링 후 검색. 초기화로 조건 비우기.
  • 목록: 행 우측의 ⋮ 메뉴로 상세 보기/수정/삭제.
  • 추가: 상단 추가 버튼으로 신규 생성. (생성 권한 필요)
  • 수정/삭제: 행 메뉴 또는 상세 화면에서 수행. (각각 수정/삭제 권한 필요)
  • Excel 다운로드: 현재 템플릿을 xlsx로 내려받기.
  • Excel 업로드: xlsx 일괄 동기화(미리보기 → 적용). (수정 권한 필요)

권한에 따른 표시: 생성 권한이 없으면 추가 버튼이, 수정/삭제 권한이 없으면 해당 행 메뉴 항목이 보이지 않습니다.

4.2 필요시험 조회 (test-match)

제품코드(mcode)를 입력하면 타이어 속성을 도출하여 수행해야 할 시험 목록을 표시합니다. 조회한 mcode는 보고서 탭과 공유됩니다.

4.3 현황 분석 (Analytics)

기준 마스터 데이터의 집계 통계를 지표로 확인합니다.

4.4 보고서 (Reports)

조회한 mcode 기준으로 보고서 형태의 결과를 확인합니다.

6. 접근 권한 관리 (관리자 전용)

접근 권한 관리 모듈은 관리자 권한이 있는 사용자에게만 표시되며, 권한 설정과 사용자 관리 두 탭으로 구성됩니다.

6.1 권한 체계 이해하기

권한은 역할(Role) 단위로 부여되며, 각 사용자에게는 하나의 역할이 배정됩니다.

메뉴 단계예시
모듈시험 항목 기준 마스터, 접근 권한 관리 …
탭기준정보 관리, 필요시험조회, 현황분석, 보고서
액션의미
조회(View)메뉴를 보고 데이터를 조회
생성(Create)신규 데이터 추가
수정(Update)기존 데이터 수정 · Excel 업로드
삭제(Delete)데이터 삭제

조회 권한이 없는 메뉴는 사이드바/탭에 아예 표시되지 않습니다. 생성·수정·삭제 권한은 화면 내 버튼 표시 여부와 서버 처리 가능 여부를 함께 제어합니다.

6.2 기본 제공 역할

역할설명권한 요약
ADMIN (관리자)시스템 관리자전 메뉴·전 액션 (접근 권한 관리 포함)
EDITOR (편집자)데이터 관리 담당데이터 메뉴 조회·생성·수정·삭제 (권한 관리 제외)
VIEWER (조회자)열람 전용데이터 메뉴 조회만

ADMIN은 시스템 역할로 보호되어 삭제할 수 없습니다.

6.3 권한 설정 (역할 × 메뉴 매트릭스)

접근 권한 관리 → 권한 설정 탭.

  1. 좌측 역할 목록에서 편집할 역할을 선택합니다.
  2. 우측 매트릭스에서 메뉴별 조회/생성/수정/삭제 체크박스를 조정합니다. (모듈 행과 들여쓰기된 탭 행으로 구성)
  3. 저장을 클릭하면 즉시 반영됩니다. 변경 권한은 해당 사용자의 다음 동작/새로고침부터 바로 적용됩니다.

역할 추가 / 삭제

  • 추가: 좌측 하단에서 역할 ID(영문 대문자/숫자/밑줄)와 이름 입력 후 역할 추가.
  • 삭제: 역할의 휴지통 아이콘. 단, 시스템 역할(ADMIN) 또는 사용 중인 역할(해당 역할 사용자가 있는 경우)은 삭제할 수 없습니다.

6.4 사용자 관리

접근 권한 관리 → 사용자 관리 탭.

  • 사용자 생성: 상단 입력행에 아이디·이름·팀·초기 비밀번호 입력, 역할 선택 후 추가.
  • 역할 변경: 목록의 역할 드롭다운에서 선택 시 즉시 반영.
  • 사용 여부 토글: 사용/중지 배지 클릭. 중지된 계정은 로그인 불가.
  • 비밀번호 재설정: 열쇠 아이콘 → 새 비밀번호 입력.
  • 삭제: 휴지통 아이콘. 기본 관리자(admin) 계정은 삭제 불가.

7. 권한 시나리오 예시

예: 특정 팀에게 "기준정보 조회만" 허용하기

  1. 권한 설정에서 역할 추가 → 예: QA_VIEW / "QA 조회".
  2. 매트릭스에서 시험 항목 기준 마스터와 기준정보 관리의 조회만 체크 → 저장.
  3. 사용자 관리에서 사용자를 생성하고 역할을 QA_VIEW로 지정.
  4. 그 사용자는 로그인 시 기준정보 관리 탭만 보이고, 추가/수정/삭제 버튼은 표시되지 않습니다.

8. 자주 묻는 질문 (FAQ)

Q. 로그인했는데 메뉴가 하나도 안 보입니다.
A. 배정된 역할에 조회 권한이 없는 상태입니다. 관리자에게 권한 부여를 요청하십시오.

Q. 버튼(추가/수정/삭제)이 보이지 않습니다.
A. 해당 액션 권한이 없기 때문입니다. 조회만 가능한 역할일 수 있습니다.

Q. 갑자기 로그아웃되었습니다.
A. 토큰 유효기간이 만료되었거나 계정이 비활성 처리되었을 수 있습니다. 다시 로그인하거나 관리자에게 문의하십시오.

Q. 권한을 바꿨는데 사용자 화면에 반영이 안 됩니다.
A. 해당 사용자가 화면을 새로고침하면 즉시 반영됩니다.

9. 운영자(설치/관리) 참고

시스템 도입·운영 담당자를 위한 항목입니다. 일반 사용자는 건너뛰셔도 됩니다.

9.1 인증/권한 데이터 테이블

인증·권한 정보는 다음 4개 테이블에 저장됩니다 (접두사 TMDM_).

테이블용도
TMDM_ROLE역할 마스터
TMDM_USER사용자 마스터 (비밀번호 bcrypt 해시)
TMDM_MENU메뉴(모듈/탭) 카탈로그
TMDM_ROLE_MENU_PERM역할 × 메뉴 권한 매트릭스

9.2 초기 시드

설치 시 ADMIN_PASSWORD 환경변수로 초기 관리자 비밀번호를 직접 지정하십시오. (지정하지 않으면 스크립트의 기본값이 사용되므로, 운영 환경에서는 반드시 지정하고 최초 로그인 후 변경하십시오.)

cd apps/api
ADMIN_PASSWORD='강력한_비밀번호' node scripts/seed-auth.cjs

9.3 환경 변수 (apps/api/.env)

변수설명
JWT_SECRETJWT 서명 비밀키 — 운영 환경에서는 반드시 강력한 무작위 값
JWT_EXPIRES_IN토큰 유효기간 (기본 8h)
AUTH_PROVIDER인증 소스: local(자체 계정) — 추후 sso(그룹웨어) 확장 예정

9.4 향후 확장 (SSO)

현재 인증은 자체 계정(local) 방식이며, 그룹웨어 SSO 연동으로 교체할 수 있도록 인증 모듈이 추상화되어 있습니다. SSO 전환 시에도 역할·권한 체계와 관리자 화면은 그대로 사용합니다.
