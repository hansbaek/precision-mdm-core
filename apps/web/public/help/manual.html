1 시스템 개요

T:MDM은 타이어 시험 기준·분류 마스터 데이터를 관리하고, 제품코드(mcode) 기반으로 수행해야 할 시험을 조회하는 웹 애플리케이션입니다. 데이터 변경에는 승인 워크플로가 적용되어, 검증되지 않은 변경이 바로 반영되지 않도록 통제합니다.

  • 접속 방식: 웹 브라우저 (로그인 후 사용)
  • 인증: 아이디/비밀번호 로그인 (JWT 토큰 기반)
  • 접근 제어: 역할(Role) 기반으로 메뉴·기능 단위 권한을 제어
  • 변경 통제: 비승인권자의 변경은 승인 대기 후 반영 (4-eyes)

2 로그인 / 로그아웃

2.1 로그인

  1. 브라우저에서 시스템 주소로 접속하면 로그인 화면(/login)으로 이동합니다.
  2. 아이디와 비밀번호를 입력합니다.
  3. (선택) 아이디 저장을 체크하면 다음 접속 시 아이디가 자동 입력됩니다.
  4. 로그인 버튼을 클릭합니다.

로그인에 성공하면 본인에게 허용된 메뉴만 표시된 메인 화면으로 이동합니다. 아이디 또는 비밀번호가 틀리면 화면에 오류 메시지가 표시됩니다.

2.2 최초 관리자 계정

시스템 최초 설치 시 기본 관리자 계정(admin)이 제공됩니다. 초기 비밀번호는 시스템 관리자(설치 담당자)에게 별도로 문의하십시오.

⚠️

보안 주의 — 최초 로그인 후 프로필 메뉴에서 비밀번호를 반드시 변경하십시오.

2.3 비밀번호 변경

우측 상단 사용자 영역 클릭 → 비밀번호 변경. 현재 비밀번호 확인 후 새 비밀번호(8자 이상)를 설정합니다.

2.4 세션 유지 / 로그아웃

  • 로그인 상태는 브라우저에 토큰으로 저장되어 새로고침해도 유지됩니다.
  • 토큰 유효기간(기본 8시간)이 지나면 자동 로그아웃됩니다.
  • 로그아웃: 우측 상단 사용자 영역 → 로그아웃.

4 시험 항목 기준 마스터

시험 항목 기준 마스터 모듈은 다음 탭으로 구성됩니다.

4.1 기준정보 관리 (대시보드)

시험 기준 템플릿(TEMPLATE_STD_TEST_ITEM)을 조회·관리합니다.

  • 검색/필터: 제품라인·검색어·시장 코드로 필터링 후 검색. 초기화로 조건 비우기.
  • 목록: 행 우측의 ⋮ 메뉴로 상세 보기/수정/삭제. 상단에서 행 밀도·페이지 크기 조정.
  • 추가 / 수정 / 삭제: 각각 생성·수정·삭제 권한 필요.
  • Excel 다운로드 / 업로드: 템플릿 xlsx 내려받기 / 일괄 동기화(미리보기→적용, 수정 권한 필요).
⚠️

승인 워크플로 적용 — 승인 권한이 없는 사용자가 추가/수정/삭제하면 즉시 반영되지 않고 승인 대기로 제출됩니다. 자세한 내용은 5장을 참고하십시오. (단, Excel 일괄 업로드는 승인 대상이 아니며 수정 권한자에게 바로 반영됩니다.)

ℹ️

생성 권한이 없으면 추가 버튼이, 수정/삭제 권한이 없으면 해당 행 메뉴가 보이지 않습니다.

4.2 필요시험 조회 (test-match)

제품코드(mcode)를 입력하면 타이어 속성을 도출해 수행해야 할 시험 목록을 표시합니다. 조회한 mcode는 보고서 탭과 공유됩니다.

4.3 현황 분석 (Analytics)

기준 마스터 데이터의 집계 통계를 지표로 확인합니다.

4.4 보고서 (Reports)

조회한 mcode 기준으로 보고서 형태의 결과를 확인합니다.

4.5 승인 관리 (Approvals)

변경요청의 제출·검토 화면입니다. 모든 사용자에게 보이며, 승인 권한 여부에 따라 표시가 달라집니다. → 5장

5 변경 승인 워크플로

데이터 무결성을 위해, 시험 항목 기준 마스터의 생성·수정·삭제는 4-eyes(이중 검토) 통제를 따릅니다. 변경한 사람과 승인한 사람이 분리되어, 검토 없이 데이터가 반영되지 않습니다.

5.1 누가 승인하나

승인 권한은 역할에 부여하는 승인 액션으로 결정됩니다(9장).

  • 승인 권한 있음: 본인 변경은 즉시 반영되고, 타인의 변경요청을 검토(승인/반려)할 수 있습니다.
  • 승인 권한 없음: 변경 시 승인 대기로 제출되며, 승인권자가 승인해야 반영됩니다.

5.2 변경 흐름

  1. 제출 — 비승인권자가 항목을 추가/수정/삭제하면 라이브 데이터는 그대로 두고 변경요청이 대기(PENDING) 상태로 등록됩니다. "승인 대기로 제출되었습니다" 안내가 표시됩니다.
  2. 알림 — 승인 권한이 있는 사용자들에게 "새 변경요청 대기" 알림이 전달됩니다.
  3. 검토 — 승인권자가 승인 관리 탭의 승인 대기 목록에서 변경 내용을 확인합니다.
  4. 승인 / 반려 — 승인하면 변경이 실제 데이터에 반영되고, 반려하면 폐기됩니다. 어느 경우든 요청자에게 결과 알림이 전달됩니다.

5.3 승인 관리 탭 사용법

승인 대기 (승인권자에게만 표시)

  • 대기 중인 변경요청이 작업 유형(생성/수정/삭제) 배지와 함께 나열됩니다.
  • 변경 내용을 펼치면 실제로 바뀌는 값을 확인할 수 있습니다.
    • 수정: 바뀐 필드만 이전 → 이후 형태로 비교 표시
    • 생성: 새로 입력된 값 전체
    • 삭제: 삭제될 대상 항목 정보
  • 내용 확인 후 승인 또는 반려(사유 입력 가능)를 누릅니다.
✅

요약만 보고 승인하지 마십시오 — 항상 변경 내용을 펼쳐 무엇이 바뀌는지 확인한 뒤 처리하십시오.

내 변경요청 (전원 표시)

  • 본인이 제출한 변경요청을 최신순 최대 50건까지, 상태(대기/승인됨/반려됨)와 함께 확인합니다.
  • 본인 요청은 본인이 승인할 수 없습니다(승인 버튼은 "승인 대기" 영역에만 표시).
  • 반려된 경우 검토자가 남긴 사유가 함께 표시됩니다.

6 알림

헤더의 알림 🔔은 본인과 관련된 비동기 이벤트를 모아 보여줍니다. (행동 직후 잠깐 떴다 사라지는 토스트와 달리, 알림은 쌓여 있다가 나중에 확인할 수 있습니다.)

6.1 사용 방법

  • 안읽음이 있으면 종 아이콘에 점이 표시되고, 패널 상단에 안읽음 개수가 나옵니다.
  • 알림을 클릭하면 읽음 처리되고, 변경요청 알림이면 승인 관리 탭으로 이동합니다.
  • 모두 읽음으로 한 번에 정리할 수 있습니다.
  • 알림은 약 30초 주기로 자동 갱신됩니다.

6.2 알림 종류

유형수신자시점
변경요청 대기승인권자비승인권자가 변경을 제출했을 때
변경요청 승인됨요청자승인권자가 승인했을 때
변경요청 반려됨요청자승인권자가 반려했을 때(사유 포함)
시스템 상태본인서버/DB 장애·복구 시 (설정에서 켠 경우)

7.2 알림

ℹ️

표시 기본값은 다음 로딩 또는 모듈 전환 시 적용됩니다. 기본값 복원으로 초기화할 수 있습니다.

8 시험 분류 마스터

시험 분류 마스터 모듈에서 시험 분류 체계를 조회·관리합니다.

9 접근 권한 관리 (관리자 전용)

접근 권한 관리 모듈은 관리자에게만 표시되며, 권한 설정과 사용자 관리 탭으로 구성됩니다.

9.1 권한 체계 이해하기

권한은 역할(Role) 단위로 부여되며, 각 사용자에게 하나의 역할이 배정됩니다.

액션의미
조회 View메뉴를 보고 데이터를 조회
생성 Create신규 데이터 추가
수정 Update기존 데이터 수정 · Excel 업로드
삭제 Delete데이터 삭제
승인 Approve변경요청 직접 반영 + 타인 요청 승인/반려
ℹ️

조회 권한이 없는 메뉴는 사이드바/탭에 표시되지 않습니다. 생성·수정·삭제·승인 권한은 화면 버튼 표시와 서버 처리 가능 여부를 함께 제어합니다.

9.2 기본 제공 역할

역할설명권한 요약
ADMIN시스템 관리자전 메뉴·전 액션 (권한 관리 포함)
EDITOR데이터 관리 담당데이터 메뉴 조회·생성·수정·삭제
VIEWER열람 전용데이터 메뉴 조회만
ℹ️

ADMIN은 시스템 역할로 보호되어 삭제할 수 없습니다. 승인 권한은 별도 액션이므로, 승인권자로 지정할 역할에 명시적으로 부여해야 합니다.

9.3 권한 설정 (역할 × 메뉴 매트릭스)

  1. 좌측 역할 목록에서 편집할 역할을 선택합니다.
  2. 우측 매트릭스에서 메뉴별 조회/생성/수정/삭제/승인 체크박스를 조정합니다.
  3. 저장하면 즉시 반영되며, 해당 사용자의 다음 동작/새로고침부터 적용됩니다.
✅

승인권자 지정 — 시험 항목 기준 마스터의 기준정보 관리 메뉴 행에서 승인을 체크하면, 그 역할 사용자는 검토 권한을 갖고 본인 변경은 즉시 반영됩니다.

역할 추가 / 삭제

  • 추가: 역할 ID(영문 대문자/숫자/밑줄)와 이름 입력 후 역할 추가.
  • 삭제: 휴지통 아이콘. 단, 시스템 역할(ADMIN) 또는 사용 중인 역할은 삭제 불가.

9.4 사용자 관리

  • 사용자 생성: 아이디·이름·팀·초기 비밀번호 입력, 역할 선택 후 추가.
  • 역할 변경: 역할 드롭다운에서 선택 시 즉시 반영.
  • 사용 여부 토글: 사용/중지 배지 클릭. 중지 계정은 로그인 불가.
  • 비밀번호 재설정: 열쇠 아이콘 → 새 비밀번호 입력.
  • 삭제: 휴지통 아이콘. 기본 관리자(admin)는 삭제 불가.

10 권한 시나리오 예시

예 1 — 특정 팀에게 "기준정보 조회만" 허용

  1. 권한 설정에서 역할 추가 → 예: QA_VIEW / "QA 조회".
  2. 매트릭스에서 시험 항목 기준 마스터·기준정보 관리의 조회만 체크 → 저장.
  3. 사용자 관리에서 사용자 생성 후 역할을 QA_VIEW로 지정.
  4. 그 사용자는 기준정보 관리 탭만 보이고, 추가/수정/삭제 버튼은 표시되지 않습니다.

예 2 — 변경은 가능하되 "승인 후 반영"되게 하기

  1. 현업 담당 역할(예: EDITOR)에 생성·수정·삭제는 주되 승인은 주지 않습니다.
  2. 팀 리더 역할에 승인을 부여합니다.
  3. 이제 담당자의 변경은 승인 대기로 제출되고, 리더가 승인 관리 탭에서 내용을 확인 후 반영합니다.

11 자주 묻는 질문 (FAQ)

Q. 로그인했는데 메뉴가 하나도 안 보입니다.
A. 배정된 역할에 조회 권한이 없는 상태입니다. 관리자에게 권한 부여를 요청하십시오.

Q. 수정했는데 "승인 대기로 제출되었습니다"라고만 뜨고 목록에 안 바뀝니다.
A. 정상입니다. 승인 권한이 없는 계정의 변경은 승인 후 반영됩니다. 승인 관리 → 내 변경요청에서 상태를 확인하세요.

Q. 승인 관리에 들어갔는데 승인 버튼이 없습니다.
A. 승인 버튼은 승인 권한이 있는 역할에게만, 그리고 승인 대기 영역에만 표시됩니다. 본인이 올린 요청은 본인이 승인할 수 없습니다. 관리자에게 승인 권한 부여를 요청한 뒤 다시 로그인하세요.

Q. 무엇이 바뀌는지 모르고 승인할 수는 없습니다.
A. 각 변경요청 카드의 변경 내용을 펼치면 수정은 이전→이후 비교, 생성은 신규 값, 삭제는 대상이 표시됩니다.

Q. 갑자기 로그아웃되었습니다.
A. 토큰 만료 또는 계정 비활성 처리일 수 있습니다. 다시 로그인하거나 관리자에게 문의하십시오.

Q. 권한을 바꿨는데 사용자 화면에 반영이 안 됩니다.
A. 해당 사용자가 새로고침(또는 재로그인)하면 즉시 반영됩니다.

12 운영자(설치/관리) 참고

🛠️

시스템 도입·운영 담당자를 위한 항목입니다. 일반 사용자는 건너뛰셔도 됩니다.

12.1 데이터 테이블

인증·권한·승인·알림 정보는 다음 테이블에 저장됩니다 (접두사 TMDM_).

테이블용도
TMDM_ROLE역할 마스터
TMDM_USER사용자 마스터 (비밀번호 bcrypt 해시, 환경설정 JSON 포함)
TMDM_MENU메뉴(모듈/탭) 카탈로그
TMDM_ROLE_MENU_PERM역할 × 메뉴 권한 매트릭스 (조회/생성/수정/삭제/승인)
TMDM_CHANGE_REQUEST변경요청(승인 워크플로) 저장
TMDM_NOTIFICATION인앱 알림

12.2 스키마 적용 (중요)

ORM 자동 동기화가 꺼져 있어, 승인/알림 기능 도입 시 아래 스키마를 운영 DB에 수동 적용해야 합니다. (스크립트: apps/api/db/2026-06-18_approval-workflow-and-notifications.sql)

-- 승인 액션 플래그
ALTER TABLE TMDM_ROLE_MENU_PERM ADD (CAN_APPROVE_YN VARCHAR2(1) DEFAULT 'N' NOT NULL);
-- + TMDM_CHANGE_REQUEST, TMDM_NOTIFICATION 테이블 (스크립트 참조)
-- 사용자 환경설정 컬럼
ALTER TABLE TMDM_USER ADD (PREFERENCES VARCHAR2(2000) NULL);
⚠️

스키마 적용 후, 권한 설정 매트릭스에서 승인권자 역할에 승인을 체크해야 승인 워크플로가 동작합니다.

12.3 초기 시드

설치 시 ADMIN_PASSWORD 환경변수로 초기 관리자 비밀번호를 지정하십시오.

cd apps/api
ADMIN_PASSWORD='강력한_비밀번호' node scripts/seed-auth.cjs

12.4 환경 변수 (apps/api/.env)

변수설명
JWT_SECRETJWT 서명 비밀키 — 운영 환경에서는 강력한 무작위 값
JWT_EXPIRES_IN토큰 유효기간 (기본 8h)

12.5 향후 확장 (SSO)

현재 인증은 자체 계정(local) 방식이며, 그룹웨어 SSO 연동으로 교체할 수 있도록 인증 모듈이 추상화되어 있습니다. SSO 전환 시에도 역할·권한·승인 체계와 관리자 화면은 그대로 사용합니다.
